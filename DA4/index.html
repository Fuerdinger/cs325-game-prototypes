Who Killed Dr. Bob? A Mystery Game

  • Originality: My game uses the visual novel genre to feature an original story with puzzle-like gameplay elements towards the end. Each interrogation sequence has a drastically different mood, depending on the relationships of the characters in the particular scene.
  • Technical Merit: My code is really small yet the game is quite lengthy. I do this using .json files to represent menu options, and I also use .json files for each set of dialogue.
  • Prototype Postmortem: I learned how to create menus that read options from .json files, as well as use "scenes" in Phaser 3 (called "states" in Phaser 2) which helped manage the different sections of my game.
  • Prototype Assets: All assets were borrowed from online.

  • Prototype Closest Other Game: I did not use anyone else's code or start with any templates. However, I did base my scene structure on https://www.phaser.io/examples/v3/view/scenes/changing-scene. My game is most similar to Phoenix Wright: Ace Attorney, a game which is mostly a visual novel (like mine) but ocassionally blends in puzzle elements to keep the player engaged. Like my game, Phoenix Wright focuses on murder mysteries.
  • High Concept: Join forces with Elon Musk and interview elite suspects to find the truth behind Dr. Bob's mysterious death!
  • Theme: I used the "celebrity sighting" storymatic theme.
  • Mandated Variety:
    Input: Keyboard
    Randomness: deterministic
    Play style: Storyteller
    Genre: visual novel / puzzle
  • Prototype Goal: The gameplay mechanic this prototype is evaluating is communcation with NPC's via branching menus/dialogue trees. As a result the actual gameplay is rather dry, but having effective menus is still a really important "mechanic" for a game.
  • Player Experience Goals: I want players to get emotionally invested in the riveting plot and play the game multiple times to see all the endings. Humor is a major focus of the game as well, and I want players to laugh at the different dialogue and scenarios.
  • Gameplay: The only input the player has in the game is either hitting "Enter" to advance the title screen, a dialogue box, or select a menu option. W/S and Up Arrow/Down Arrow are used to navigate multiple options. One of the gameplay mechanics I had planned was an ability to choose the questions you ask suspects, which could result in more information being exposed or less information being exposed.
  • Strategies: To complete the game, players will have to pay close attention to each interrogation sequence. Of course, they can still brute force the questionaire at the end, but all puzzle games suffer from this to some extent.
  • Story/Setting/Premise: The game is set in the real world. Dr. Bob (play my first digital assignment to learn more about him) is an employee at SpaceX and has been murdered by an unknown party. Elon Musk tasks you, another SpaceX employee, to track down the killer, which Elon has narrowed down to 4 rich elites.
  • Target Audience: The target audience for this game is anyone 12-18 years old. Anyone who can read can play, but the humor is targeted towards younger people, and is a major focus of the game, so I imagine older folks likely would find the game too silly.
  • Play Time: The game takes about 4 minutes to play. However, the ability of a player to skip dialogue boxes if they desire means that the game can technically be beaten in about 15 seconds, but not on the first playthrough.
